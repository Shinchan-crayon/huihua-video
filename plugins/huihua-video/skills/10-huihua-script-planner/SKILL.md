# 绘画视频口播规划

## 产品边界

仅处理当前 huihua-video 项目。不得读取或写入 Shin-video 仓库、`.shin-video-runtime` 或其历史产物。

## 职责

读取原文，保留事实、人物、因果和情绪转折，输出适合配音的口播文案。不要在此阶段决定具体动画模板。

## 输出

生成 `narration.json`：

```json
{
  "title": "视频标题",
  "language": "zh-CN",
  "full_text": "完整口播",
  "sentences": [
    {
      "id": "sentence-001",
      "text": "完整句子",
      "function": "hook",
      "visual_hint": "这句话需要解释的画面意义"
    }
  ]
}
```

`sentences[].text` 必须来自 `full_text`，顺序一致，不得把一句话截断成残句。生成后直接作为本次最终口播进入音频阶段，不展示草稿，不等待确认。

## 写作规则

- 口语化，但不制造口头禅和夸张事实。
- 每句话表达完整意思；字幕可按语义标点进一步显示，但正文始终以本次最终口播为准，
  服务商原生字幕文件只提供时间边界，不得改写正文。
- `visual_hint` 解释“画面要说明什么”，不复写整句口播。
- 明确钩子、铺垫、转折、证明、高潮和收束。
- 删除网页按钮、播放器文案、关注提示等污染内容。
