# 音频字幕契约

## 支持的音频路径

当前版本支持 MiniMax `speech-2.8-hd` 与 Doubao `seed-tts-2.0`。本次最终 `narration.json` 是配音文字和字幕正文的唯一来源；已选择服务商生成的最终音频是视频总时长、字幕时间和节奏点的唯一计时依据。

更改口播、模型、音色或语速后，旧音频、原生字幕文件、`subtitle-timeline.json` 及下游产物全部失效。

## 服务商原生时间戳

MiniMax 调用时必须设置：

```json
{
  "subtitle_enable": true,
  "subtitle_type": "word"
}
```

非流式响应必须返回 `data.subtitle_file`。该地址返回带毫秒时间戳的 JSON 字幕。

Doubao 调用时必须请求 SSE 音频，并在 `audio_params` 中设置 `enable_subtitle: true`。事件 `351` 必须返回词级时间戳，事件 `352` 返回音频分片，事件 `152` 为完成帧。

下载或整理后的原生字幕文件必须保存在视频项目内，并在 `subtitle-timeline.json` 中记录文件路径、SHA-256、格式和条目数量。MiniMax 或 Doubao 未返回有效原生字幕、字幕文字无法对应最终口播、时间区间冲突或字幕超出音频时长时，立即停止。禁止估算时间，禁止切换到其他识别工具。

## 字幕正文

- 字幕正文只来自本次最终 `narration.json`。
- 服务商原生字幕只负责提供时间边界，不得改写正文。
- 映射后按 `sentence_id` 拼接的文字必须与原口播逐句一致。
- 一条字幕优先显示一个完整句子；长句只允许在自然标点处拆分。
- 不允许半句话在没有语义停顿时突然切走。

## 时间质量

- `start` 和 `end` 使用秒。
- 条目按起始时间递增。
- 不允许负数、反向区间和字幕冲突。
- 首尾可以保留合理静音，但字幕不能超出音频时长。
- 用户要求开场停顿时，必须由音频脚本添加真实静默，并同步平移字幕和节奏点；禁止只延后画面。
- 时间轴必须保存音频 SHA-256；音频变化后门禁必须拒绝旧时间轴。

## 渲染

字幕在 Remotion 顶层独立渲染，不嵌套到场景 `Sequence` 中。字幕背景默认透明，可使用轻微文字描边或阴影保证可读性。
