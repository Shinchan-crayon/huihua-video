# 音频字幕契约

## 唯一音频路径

当前版本只支持 MiniMax TTS。已确认的 `narration.json` 是配音文字和字幕正文的唯一来源，MiniMax 生成的最终音频是视频总时长、字幕时间和节奏点的唯一计时依据。

更改口播、模型、音色或语速后，旧音频、原生字幕文件、`subtitle-timeline.json` 及下游产物全部失效。

## MiniMax 原生时间戳

调用 MiniMax 时必须设置：

```json
{
  "subtitle_enable": true,
  "subtitle_type": "word"
}
```

非流式响应必须返回 `data.subtitle_file`。该地址返回带毫秒时间戳的 JSON 字幕。下载后的原生字幕文件必须保存在视频项目内，并在 `subtitle-timeline.json` 中记录文件路径、SHA-256、格式和条目数量。

MiniMax 未返回有效原生字幕、字幕文字无法对应已确认口播、时间区间冲突或字幕超出音频时长时，立即停止。禁止估算时间，禁止切换到其他识别工具。

## 字幕正文

- 字幕正文只来自已确认的 `narration.json`。
- MiniMax 原生字幕只负责提供时间边界，不得改写正文。
- 映射后按 `sentence_id` 拼接的文字必须与原口播逐句一致。
- 一条字幕优先显示一个完整句子；长句只允许在自然标点处拆分。
- 不允许半句话在没有语义停顿时突然切走。

## 时间质量

- `start` 和 `end` 使用秒。
- 条目按起始时间递增。
- 不允许负数、反向区间和字幕冲突。
- 首尾可以保留合理静音，但字幕不能超出音频时长。
- 时间轴必须保存音频 SHA-256；音频变化后门禁必须拒绝旧时间轴。

## 渲染

字幕在 Remotion 顶层独立渲染，不嵌套到场景 `Sequence` 中。字幕背景默认透明，可使用轻微文字描边或阴影保证可读性。
