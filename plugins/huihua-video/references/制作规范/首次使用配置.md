# 首次使用配置

## 1. 首次环境配置

只有首次缺少依赖或模型配置时运行：

```bash
python3 scripts/doctor.py
```

需要 Node.js、npm、Python、FFmpeg、FFprobe、已选择的音频模型配置和 `$image-prompt-generator`。

新项目必须使用独立目录，不能建立在 `Shin-video` 仓库或 `.shin-video-runtime` 内。中间产物只放在项目内 `.huihua-video-runtime/`。

创建新项目时运行：

```bash
python3 scripts/initialize_huihua_project.py \
  --project-dir /absolute/path/to/project
```

初始化脚本只创建项目目录和 `.huihua-video-runtime/`，不创建工作流状态台账。

## 2. 风格与比例

用户指定风格或比例时直接采用。未指定时自动按内容选择古风手绘或卡通手绘，并使用默认比例 `3:4`。在视频项目根目录创建 `style-profile.json`，每条图片 Prompt 必须明确写入同一个比例。

## 3. 选择并配置音频模型

只有首次缺少默认音频模型或用户明确要求更换时，才提供以下选择：

1. `minimax-speech-2.8-hd`
2. `Doubao-语音合成-2.0`
3. 其他

选择 `minimax-speech-2.8-hd` 时运行：

```bash
python3 scripts/configure_minimax.py
```

脚本会引导打开 https://www.minimaxi.com/audio/voices，试听音色并保存完整的默认 `voice_id`。

选择 `Doubao-语音合成-2.0` 时运行：

```bash
python3 scripts/configure_volcengine.py
```

脚本会引导打开 https://console.volcengine.com/ark/region:cn-beijing/experience/voice?model=doubao-seed-tts-2-0，试听音色并保存完整的默认 `voice_id`。

两种配置脚本都会通过隐藏输入接收 API Key，并把所选服务商设为默认音频模型。选择“其他”时必须提供服务商、模型、音色 ID、调用文档及可逐字或逐词对齐的原生时间戳能力；在适配器完成测试前不得生成音频或渲染。

API Key 不得出现在聊天记录、命令参数、视频项目、交付目录或 Git 仓库中。自动化环境可使用 `MINIMAX_API_KEY` 或 `VOLCENGINE_TTS_API_KEY`，但仍需在配置文件中保存所选 `voice_id`。

## 4. 配置图片工作流

绘画视频的图片规划、提示词和生图统一调用 `$image-prompt-generator`。

尚未安装时，在 Codex 中输入：

```text
请从 https://github.com/Shinchan-crayon/image-prompt-generator 安装 image-prompt-generator Skill。
```

首次需要实际生成图片时，按照该 Skill 的引导选择图片渠道并通过隐藏输入配置该渠道自己的 API Key。图片渠道不得复用任一音频模型 API Key。

用户调用 `$huihua-video` 制作视频，即视为授权本次 Prompt 生成、付费生图和自动重试。正常制作中不展示 Prompt，不再次索要批准。

## 5. 后续制作

配置完成后直接使用默认模型制作。后续视频不重复运行环境检查；只有实际命令因缺少依赖或配置失败时才报告阻塞。
