# 首次使用配置

## 1. 检查本地环境

在插件目录运行：

```bash
python3 scripts/doctor.py
```

需要 Node.js、npm、Python、FFmpeg、FFprobe、MiniMax 配置和 `$image-prompt-generator`。

## 2. 配置 MiniMax

当前版本只支持 MiniMax TTS。运行：

```bash
python3 scripts/configure_minimax.py
```

配置脚本会：

1. 通过隐藏输入接收 MiniMax API Key。
2. 提示打开 https://www.minimaxi.com/audio/voices 试听音色。
3. 接收从试听页复制的完整 `voice_id`。
4. 将配置保存到用户配置目录，并设置为仅当前用户可读写。

API Key 不得出现在聊天记录、命令参数、视频项目、交付目录或 Git 仓库中。自动化环境可使用 `MINIMAX_API_KEY`，但仍需在配置文件中保存所选 `voice_id`。

## 3. 配置图片工作流

绘画视频的图片规划、提示词和生图统一调用 `$image-prompt-generator`。

尚未安装时，在 Codex 中输入：

```text
请从 https://github.com/Shinchan-crayon/image-prompt-generator 安装 image-prompt-generator Skill。
```

首次需要实际生成图片时，按照该 Skill 的引导选择图片渠道并通过隐藏输入配置该渠道自己的 API Key。图片渠道不得复用 MiniMax API Key。

图片 Prompt 必须先展示给用户审核；只有用户看到当前版本 Prompt 后明确批准，才能产生付费生图请求。

## 4. 重新检查

完成音频和图片配置后再次运行：

```bash
python3 scripts/doctor.py
```

只有 `ok: true` 才进入正式制作。
